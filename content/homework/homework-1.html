  1. Complete the Introduction to LINUX and C Programming tutorial that is linked from the Resources page, if you have not yet completed it.

  2. If you have not yet installed Ubuntu or another LINUX distribution, instructions for installing Ubuntu are available here. You should begin your installation as soon as possible. Your installation should be completed no later than the first Wednesday lab period.

  3. You have now gone through the steps of using Emacs to edit C program, working with the LINUX command line environment and compiling C programs from the command line. Do the following as a self-test. If you have problems doing this, the instructor and TAs are available to help you.

    During the next class your instructor may ask that you demonstrate that you have completed this assignment. For the purpose of this assignment $LINUX_HOST refers to either your laptop, if you have linux installed, or {abacus/sliderule}.csse.rose-hulman.edu.

    1. Login to $LINUX_HOST. Use SecureCRT to access abacus or sliderule if have not yet installed linux on your laptop.

    2. Create a directory called HW01.

    3. Using Emacs, create in HW01 a C program file called MyFirstCProgram.c.

    4. Edit MyFirstCProgram.c so that it prints the string ``This is my first Operating Systems C program.'' to the console. Look at the HelloWorld.c program for an example on how to do this.

    5. Save the file.

    6. Use ls to ensure that a file called MyFirstCProgram.c has been created.

    7. We will use the Google C++ style. Use cpplint.py (this version has a couple of small local modifications to make it work better for C code) to make sure your code conforms to a reasonable style. You might need to make cpplint.py executable (e.g. by running ``chmod a+x cpplint.py''). You can check foo.c by running ``/cpplint.py foo.c'' assuming cppylint.py is in your home directory. If cpplint.py is in a different directory, you'll need to include the path to it. If it's in the current directory the path to it is ``./cppylint.py.'' Note: you can place CPPLINT.cfg in the root of your directory tree to automatically load local configuration options. This config file will eliminate the copyright warning.

    8. Compile MyFirstProgram.c using gcc with and without the ``-o'' flag and create the executables ``a.out'' and ``myfirstprog''.

    9. Run each executable.

    10. When done, logout of $LINUX_HOST.

    1. Log back into to $LINUX_HOST and cd to the HW01 directory.

      > mv MyFirstCProgram.c a.out myfirstprog <svn local workspace>/HW01
